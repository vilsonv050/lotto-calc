1. Выбор лотереи
  • Восьмерки: П1 1–20, П2 1–4.
  • Премьер: П1 1–20, П2 1–20. Макс: 12 в поле, 18 суммарно.
  • ТОП-12: П1 1–24. П2 нет.
2. Исключения / Включения Исключения — приоритет №1. Включения с фильтром «Случайных N» — берёт N из списка вместо обязательных всех. 3. Правила разниц Формат "раз-кол". Чекбоксы П1/П2 для выбора поля применения. Блок правил имеет фиксированную высоту с прокруткой — добавление любого количества правил не сдвигает интерфейс. 4. Фильтры комбинаций (по билету) Эти фильтры отбрасывают неподходящие билеты из базы. Работают как в режиме обычного рандома, так и при подсчёте правил разниц.
  • Сумма (75-85): оставляет только билеты, сумма чисел которых попадает в коридор включительно.
    Пример для 8 из 20: "75-85". Билет [1,2,3,4,5,6,7,8] имеет сумму 36 — отбрасывается. Билет [5,8,10,11,13,14,17,19] имеет сумму 87 — не в коридоре, отбрасывается.
  • Разброс (12-16): разница между максимальным и минимальным числом в билете.
    Пример: "12-16". Билет [2,5,9,11,15,17,19,20] имеет разброс 18 — отбрасывается. Билет [5,8,10,12,14,15,17,19] имеет разброс 14 — проходит.
    Позволяет получать "компактные" билеты.
  • Без соседей: чекбокс. Запрещает числа идущие подряд с шагом +1.
    Пример: [3, 7, 12, 15] — проходит. [3, 4, 12, 15] — отбрасывается (3 и 4 соседние).
  • Секторы (1-10:3, 11-20:5): жёсткие квоты по диапазонам. Сумма количеств = числам в поле.
    Пример для 8 чисел: "1-10:3, 11-20:5". Билет содержит ровно 3 из 1-10 и ровно 5 из 11-20. Билет [1,2,3,4,5,11,12,13] — отбрасывается (5 из 1-10 вместо 3).
5. Пакетные фильтры (связь между билетами)
  • Связь N: переставляет билеты в пачке так, чтобы каждый имел ровно N общих чисел с предыдущим.
    N=2. Билет 1: [3,7,10,15]. Билет 2 подбирается с ровно 2 общими числами (например, [3,10,12,18]). Если подходящего нет — ставится любой, в статусе пишется количество сбоев.
  • Покрытие ≥: информационный анализ — проверяет, что каждое число поля встретилось ≥ N раз.
    Покрытие ≥ 3. Если число 14 встретилось 1 раз — предупреждение: "⚠️ Покрытие: 14(1) < 3". Не отсекает билеты, только анализирует.
6. Сочетание фильтров Все 6 фильтров работают одновременно. При правилах разниц фильтры по билету встраиваются в перебор — кнопка "Посчитать" покажет количество с их учётом. При заморозке П1 — только к П2. 7. Заморозить П1 Первая генерация запоминает П1. Последующие — берут тот же П1, пересчитывают только П2. 8. Алгоритмы выборки Описание по кнопке «?».
