Генератор Разниц [ 86 ]
АЛГОРИТМ ВЫБОРКИ (по умолчанию: Классический)
?Равномерное распределение

Всю найденную базу комбинаций скрипт делит на ровные куски (по количеству запрошенных билетов). Из каждого куска берётся ровно 1 случайный билет.

Результат: билеты будут равномерно «размазаны» по всей базе.

Когда полезно: если хотите максимальное разнообразие по структуре билетов.

Минус: не учитывает пересечения чисел между билетами.
?Анти-пересечение

Первый билет берётся случайно. Каждый следующий подбирается так, чтобы иметь минимум общих чисел со ВСЕМИ уже выбранными билетами.

Как работает: из оставшихся кандидатов проверяется до 100 вариантов, выбирается тот, чьё пересечение с общим набором выбранных чисел минимально.

Когда полезно: когда нужно покрыть максимум разных чисел из поля.

Минус: билеты могут получиться с неестественными суммами.
?Взвешенный по сумме

Вычисляется «золотая середина» — математическое среднее суммы: N × (MaxNum + 1) / 2.

Все комбинации сортируются по близости суммы к этому значению. Из лучших 20% берётся случайная выборка.

Когда полезно: если хотите исключить экстремально «лёгкие» или «тяжёлые» билеты.

Минус: сужает разнообразие, все билеты имеют похожие суммы.
?Интеллектуальный поток (Reservoir Sampling)

Классический алгоритм из теории вероятностей. Скрипт «проходит» по всей базе и на каждом шаге подбрасывает монетку — заменить ли текущий билет в выдаче.

Математическое свойство: каждая комбинация имеет абсолютно равную вероятность попасть в выборку.

Когда полезно: когда нужна математически честная выборка без перекосов.

Минус: результат неотличим от обычного рандома по виду.
?Историческая эксклюзия

Запоминает КАЖДЫЙ скачанный билет в памяти браузера. При новой генерации все ранее выгруженные билеты исключаются.

Как работает: фильтруются билеты из базы. Если уникальные билеты исчерпаны — берутся случайные (с повторами).

Когда полезно: для серии выгрузок — гарантирует отсутствие дублей между выгрузками.

Минус: пул со временем исчерпается. Сброс — кнопка «Очистить всё».

  • Фильтр «Случайных N из включений» — чекбокс и поле N рядом с включениями
  • Чекбоксы «Поле 1» / «Поле 2» в правилах разниц
  • Дедупликация, авто-подсчёт, анти-пересечение со всеми
Общий принцип работы Скрипт — генератор лотерейных билетов с двумя режимами: обычный рандом и генерация по математическим шагам (разницам). 1. Выбор лотереи
  • Восьмерки: Поле 1 — 1–20. Поле 2 — 1–4.
  • Премьер: Поле 1 — 1–20. Поле 2 — 1–20. Лимиты: макс. 12 в поле, 18 суммарно.
  • ТОП-12: Поле 1 — 1–24. Поле 2 отсутствует.
2. Исключения Указанные числа никогда не появятся в билете. Приоритет №1 — фильтруют и основной пул, и список включений. 3. Включения и фильтр «Случайных N» Рядом с полем включений есть чекбокс и маленькое поле для цифры N.
  • Чекбокс снят (по умолчанию): стандартное поведение. Все введённые числа обязательны в КАЖДОМ билете.
  • Чекбокс включён: список включений становится «базой-кандидатом». Для каждого билета скрипт случайно берёт ровно N чисел из этого списка, а остальные слоты заполняет рандомом из общего пула.
  • Пример: Включения: "1, 2, 3, 4". N = 2. В первом билете выпадет [2, 4] + 6 рандомных. Во втором — [1, 3] + 6 других рандомных.
  • Фильтр работает в сочетании со всем остальным: исключения вычитаются из кандидатов; при правилах разниц фильтруются только те комбинации, где ровно N чисел из списка; алгоритмы выборки применяются к отфильтрованной базе.
  • Ограничения: N не может быть больше размера списка включений (после вычета исключений) и больше количества чисел в поле.
4. Правила разниц Разница — шаг между соседними числами. Формат: "раз-кол" (напр. "1-3, 5-1"). Под строкой ввода — чекбоксы выбора поля (Поле 1 / Поле 2). Для двухполевых лотерей можно применять разницы к любому полю или к обоим сразу. 5. Алгоритмы выборки Решают, какие именно билеты из базы попадут в выгрузку. Описание по кнопке «?». 6. Выгрузка CSV (UTF-8 BOM), разделитель полей — точка с запятой, чисел — пробел. Файл: "lottery_export.csv". 7. Кнопка «Очистить всё» Полный сброс: лотерея → Восьмерки, все поля пусты, алгоритмы сброшены, правила удалены, история стёрта.
